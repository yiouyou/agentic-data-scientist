# 当实现声明某部分不可行时怎么办？

若实现摘要表明某些部分不可行或需替代方案，请采取建设性处理：

1. **承认挑战**：认可实现中发现的真实技术约束

2. **双向评估**：
   - 评估编码智能体顾虑的合理性
   - 评估原计划是否忽略了技术现实
   - 寻找折中且可执行的替代路径

3. **记录分析过程**：给出平衡、可追溯的评估结论

4. **建设性回应示例**：
   - “编码智能体遇到库 X 的依赖冲突。可评估更新环境，或改用库 Y 实现类似目标。”
   - “实现发现数据格式与计划假设不一致。在这些约束下，采用格式 Z 的替代方案是合理的。”

请记住，目标是协作式问题解决：在承认现实约束的同时，找到保持分析严谨性的最佳前进路径。
